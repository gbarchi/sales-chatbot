1. ¿Qué es la aplicación?

AskJaime (internamente Maviju Salesbot) es un chatbot de analítica de ventas impulsado por IA. Permite que cualquier usuario del área comercial de MAVIJU pregunte en lenguaje natural —en español— sobre los datos de ventas, y recibe como respuesta una tabla, un gráfico interactivo y una explicación redactada por el modelo.

El corazón del sistema traduce la pregunta del usuario a SQL mediante el modelo Claude, la ejecuta contra una base analítica DuckDB cargada desde el CSV de facturación de la empresa, y decide automáticamente el tipo de visualización más adecuado. Todo respeta el alcance por rol (un vendedor solo ve sus datos) y las restricciones de margen (solo gerencia ve rentabilidad).

🎯 Objetivo

Responder "¿cuánto vendí?", "¿quién es mi mejor cliente?", "¿cómo va la categoría X?" sin saber SQL ni abrir un dashboard.

👥 Para quién

Administradores, gerencia, supervisores de ventas y vendedores (FFVV).

⚙️ Cómo

Pregunta en español → Claude genera SQL → DuckDB ejecuta → gráfico + análisis, con filtros por rol.

3. Arquitectura y unidades desplegables

El backend mantiene dos bases separadas: DuckDB (datos de ventas, solo lectura, en memoria) para la analítica y SQLite para usuarios, historial, favoritos, logs y feedback.

3.1 Arranque del servidor

Al iniciar, index.js ejecuta en orden: 1) inicializa SQLite (crea tablas y usuarios por defecto si la BD está vacía), 2) inicializa el servicio LLM (Claude), y 3) carga el CSV en DuckDB (≈2M filas, tarda de 10 a 30 s). Solo después el servidor acepta consultas. El endpoint /api/health reporta initialized para los healthchecks.

6. Ciclo de vida de una consulta

El núcleo funcional es la traducción de lenguaje natural a SQL. Cada pregunta atraviesa una tubería con controles de seguridad, resolución de entidades y auto-corrección de SQL.

6.1 Puntos clave de la tubería

  • Rate limiting: 30 consultas de chat/min y 120 generales/min por IP (las llamadas al LLM son las caras).
  • Pre-check de margen: si el usuario sin permiso menciona "margen", "ganancia", "utilidad" o "rentabilidad", se bloquea antes de llegar a Claude.
  • Resolución de entidades: entityResolver.js normaliza (minúsculas + sin acentos) y hace match por palabra para nombres de persona: "Ronny" → "Ronny Marcillo".
  • Multi-consulta: preguntas complejas pueden disparar varias SQL (p. ej. una barra + un plan de visitas), combinadas en una sola respuesta.
  • Auto-corrección: si el SQL falla, se reintenta corrigiéndolo; si sigue fallando, se relajan los filtros.
  • Alcance por rol: a cada SQL se le inyecta el filtro del usuario (ver §7).

7. Usuarios, roles y permisos

Autenticación por JWT (token + cookie), contraseñas con bcrypt. Cada rol determina qué filas ve el usuario y si puede consultar el margen. Existe además un cierre de sesión por inactividad.

RolQué ve¿Margen?
adminTodos los datos + panel de administración (usuarios, logs, settings, feedback).✅ Sí
gerenteAcceso completo a todos los datos de ventas.✅ Sí
supervisorSolo su equipo — filtro NombreSupervisor = <nombre>.❌ No
vendedorSolo sus propias ventas — filtro Slpcode = <código>.❌ No
Protección de margen en 4 capas: (1) pre-check por intención antes del LLM, (2) ocultar columnas de costo/margen del esquema mostrado a Claude, (3) reglas y ejemplos de margen omitidos del prompt, (4) validación del SQL generado antes de ejecutarlo. Rol desconocido → sin acceso a margen (falla cerrado).

7.2 Administración

El panel de admin (solo rol admin) ofrece CRUD de usuarios y reseteo de contraseña, configuración de la API key y el modelo de Claude en caliente (llmService.reinitialize), visor de logs de consultas con estadísticas, y resumen de feedback. Los vendedores se asocian a su slpcode; los supervisores a un supervisor_name.

Atajos (hoy, semana, mes, año) y rangos personalizados aplicados a Fecha.

⭐ Favoritos e historial

Historial de preguntas por fecha y consultas guardadas reutilizables.

📥 Exportar

Descarga de resultados como CSV y tabla de datos plana.

🗺️ Planificador de visitas

Vista plan: clientes rankeados en tarjetas con barra de progreso hacia una meta.

📉 Churn / perfil de cliente

Vistas churn y profile para clientes en riesgo y ficha 360° de un cliente.

👍 Feedback

Valoración de cada respuesta con corrección opcional para mejorar la calidad.

# Desarrollo local
cd server && npm install && npm start    # API en :3001
cd client && npm install && npm start    # SPA en :3000 (proxy → :3001)

# Evaluación de calidad de respuestas
